# Keenan's NixOS Config

Flake-based NixOS configuration for my machines, built on the
**dendritic pattern** ([flake-parts](https://flake.parts) +
[import-tree](https://github.com/vic/import-tree) +
[flake-file](https://github.com/vic/flake-file)).

> This is my own setup. It's opinionated and tailored to my hardware and preferences.
> It's meant as a reference; not necessarily something for you to deploy as-is.
> Please take what works for you! I've added comments and links
> to explain 'why' I do things a certain way.

My previous, non-dendritic config is located in the `main` branch of this repo. I may go back!

## Overview

- My primary use-case is gaming, so a majority of the work went into making that work how I want it.
- Preference on bleeding edge software when (easily) possible. Mesa-git and other nightly builds.
- I tinker a lot, so I'm constantly experimenting with various settings.

## Features

- **Dendritic modules** -- features as composable `nixos`/`homeManager` aspects.
- **Impermanence** -- `tmpfs` root, persistent state on `btrfs` subvolumes,
  declared via the [preservation](https://github.com/nix-community/preservation)
  module and [disko](https://github.com/nix-community/disko).
- **Secrets** with [sops-nix](https://github.com/Mic92/sops-nix)
- **Secure Boot** support (`sbctl`) and reproducible bare-metal installs via
  [nixos-anywhere](https://github.com/nix-community/nixos-anywhere) +
  [nixos-facter](https://github.com/nix-community/nixos-facter).
- **Home Manager**, **nix-flatpak**, **plasma-manager**, and tons of others.
- **`Justfile`** wrapping the day-to-day and provisioning workflows.

## Repo structure

```
.
├── flake.nix            # generated by flake-file -- do not edit
├── Justfile             # workflow + provisioning recipes
├── .sops.yaml           # sops recipients
├── modules/             # import-tree'd
│   ├── flake/            # flake-parts wiring (import-tree, flake-file,
│   │                     # git-hooks, formatter, per-system pkgs, host builders)
│   ├── profiles/        # composite aspects (base, desktop, gaming, …)
│   ├── hosts/           # per-host modules: disko, hardware, host facts
│   ├── apps/            # per-application aspects
│   ├── users/           # user definitions
│   ├── sops/            # secrets wiring + encrypted secret files
│   └── secure-boot/     # Secure Boot (sbctl) configuration
├── packages/            # custom packages (pkgs-by-name-for-flake-parts)
├── assets/              # wallpapers, theming assets
└── docs/                # notes
```
